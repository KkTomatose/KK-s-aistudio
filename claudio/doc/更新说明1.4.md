# Claudio v1.4 更新说明

本文档汇总 v1.4 版本的所有新增功能、优化和修复。

---

## 1. 🎨 AI 电台卡片 UI 全面美化

### 整体设计升级

对 AI 电台正反面卡片进行了一次完整的视觉打磨，保持原有深色科技感配色，重点提升质感和层次。

**样式系统变化：**

- **圆角体系**：从单一 `6/8/16px` 升级为外壳大圆角、模块中圆角、控件小圆角的三段式体系
- **阴影体系**：从单层投影升级为环境投影 + 边缘阴影 + 内阴影的组合，深色主题下更显质感
- **边框体系**：外壳边框稍强、内部稍弱、选中态半透明 accent 边框，用强弱拉开信息层级
- **表面纹理**：加入极轻径向高光，只放在壳体和大区块，不干扰文字

**正面卡片改造：**

- 时钟区保留像素字体和 dot grid，ON AIR 状态发光更克制
- 播放器、队列、聊天四个 section 通过背景层次、间距和局部圆角分区
- 队列当前曲目从硬 `border-left` 改为整项微亮 + 轻发光
- 聊天欢迎态、消息气泡、输入框统一圆角和边框语言

**背面卡片改造：**

- `.flip-back` 与正面共享同一外壳逻辑（圆角、描边、投影）
- 历史面板做成更安静的"档案面"
- 历史列表信息秩序优化，hover 态更轻

**页面氛围：**

- 新增柔和环境光斑，让卡片从背景更自然地浮出
- 头部控制区（RECORD / SETTINGS / 主题切换）统一成精致控制组

**移动端：**

- 保留卡片圆角和外边距，不再直接清零

**兼容性：**

- 增加 `@supports not (background: color-mix(...))` 回退样式

## 2. 🔊 TTS 播报音链路重构

### 从"长期缓存"改为"临时资源 + 自动清理"

**v1.3 行为：** TTS 按 `文本 + 引擎` 哈希长期复用同一 mp3，`clearCache()` 定义了但未接入定时任务

**v1.4 行为：** 每次播报生成唯一 `ttsId` 和唯一 mp3 文件，播完后自动进入清理流程

### 核心改造

**唯一资源生成：**

- `tts.js` 不再按文本哈希生成文件名，改为 `{timestamp}-{随机hex}-{engine}.mp3`
- 每次生成登记到内存 `liveTts` map
- 返回结构从文件路径升级为 `{ ttsId, filePath, publicUrl }`

**前端回执机制：**

- `say` 事件新增 `ttsId` 字段
- 前端 `audioTts` 在 `ended`、`error`、被下一条播报打断时，调用 `POST /api/tts/:id/complete`
- 页面关闭/刷新时通过 `pagehide + sendBeacon` 兜底回执

**延迟删除：**

- 服务端收到回执后不立刻删，进入延迟删除窗口
- 默认 `10s` 延迟，可通过 `TTS_DELETE_DELAY_MS` 环境变量调整

**SW 缓存同步：**

- Service Worker 新增 `DELETE_CACHE` 消息处理
- 前端播完后通知 SW 删除对应 `/tts/*.mp3` 浏览器缓存

**孤儿文件兜底：**

- 新增 `pruneStaleTts()` 周期清理
- 启动时扫一次，之后每 5 分钟扫一次超过阈值的孤儿文件
- 孤儿阈值默认 `30 分钟`，可通过 `TTS_ORPHAN_MAX_AGE_MS` 调整

**重启恢复：**

- `releaseTts()` 现在不仅查内存 `liveTts`，如果内存无记录会从磁盘按文件名恢复，再进入删除流程

### 播报冲突处理

- 如果一条播报正在播放，又来一条新播报，前端不会叠播
- 上一条会按 `interrupted` 收尾，新播报直接替换

## 3. 🎵 播放逻辑增强

### 歌曲 URL 自动补链

新增 `GET /api/song/:id/url` 接口，前端可在歌曲链接失效或缺失时主动刷新。

**自动补链场景：**

- 恢复旧会话后，当前曲目的 URL 为空或过期
- 点击播放时，`audio.play()` 失败 → 自动刷新 URL 并重试
- `playPrev()` / `playNext()` 恢复播放失败时同样走刷新链路

**音频错误恢复：**

- `audioMain` 新增 `error` 监听，播放失败后自动刷新链接重试
- 增加防重试锁，避免连续错误时死循环
- 接口不可用时 toast 提示用户，不再静默失败

### 播报与音乐自动同步

新增 `autoStartMusic` 机制，确保点击"今日歌单"或发送对话后，播报与音乐同步启动。

**服务端：**

- `say` 事件新增 `autoStartMusic` 字段
- 只有会推送歌曲的链路（聊天推荐、自动续播、今日歌单、早报歌单）才下发此标记
- 纯播报不会误触发自动放歌

**前端行为：**

| 场景 | 行为 |
|------|------|
| 有暂停中的当前曲目 | 立即恢复当前曲目播放 |
| 无当前曲目 | 第一首新增歌曲到来时自动接管并播放 |
| 正在正常播放 | 不抢播、不切歌，新歌只加入队列 |

### 媒体播放权限解锁

修复刷新页面后首次点击"今日歌单"或发送消息时，播报和音乐不自动播放的问题。

**根因：** 浏览器要求媒体播放必须在用户手势的同步调用链中发起，而 `fetch → WebSocket` 是异步的，脱离手势上下文后 `audio.play()` 会被拦截。

**修复：**

- 新增 `unlockMediaPlayback()` 函数，在用户手势期间用静音音频预解锁
- 连接到"今日歌单"按钮点击、发送消息、页面首次 `pointerdown`、页面首次键盘操作
- 按钮先进入 loading 状态再解锁媒体，避免"点击后卡住再显示生成中"

## 4. 🖼️ 播放栏排版重构

### 全新双栏布局

播放栏从原来的横向单行改为**左侧封面 + 右侧信息/控制**的双栏结构。

**桌面布局：**

```
┌──────────────────────────────────────────┐
│  ┌──────────┐  歌曲名                      │
│  │          │  歌手 · 专辑名                │
│  │  封面    │  ◀ ▶ ⏯ ■  VOL ━━━━━         │
│  │          │  0:00 ━━━━━━━━━━━━━━ 3:45    │
│  └──────────┘                              │
└──────────────────────────────────────────┘
```

**封面获得方式：**

- 优先从网易云搜索/歌单接口自带的 `al.picUrl` 获取
- 若无则通过 `song/detail` 接口兜底补全
- 前端自动补当前曲目缺失的封面/专辑信息（`GET /api/song/:id/meta`）
- 封面加载失败时自动显示 fallback

**响应式适配：**

| 宽度 | 布局 |
|------|------|
| ≥800px | 左右双栏，封面列 200px |
| 600–799px | 左右双栏，封面列 140px |
| <600px | 上下堆叠，封面全宽 |

## 5. 📡 API 新增

### 新增 HTTP 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/song/:id/url` | 按网易云歌曲 ID 刷新播放链接 |
| `GET` | `/api/song/:id/meta` | 按网易云歌曲 ID 获取专辑名和封面 |

### WebSocket 事件结构更新

`say` 事件新增字段：

```json
{
  "text": "播报词",
  "audio": "/tts/xxx.mp3",
  "ttsId": "唯一播报ID",
  "autoStartMusic": true
}
```

`now-playing` 事件新增字段：

```json
{
  "name": "歌名",
  "artist": "歌手",
  "album": "专辑名",
  "cover": "封面URL",
  "url": "播放链接",
  "id": "歌曲ID",
  "dbId": 123
}
```

## 6. 🛠️ 其他修复与优化

### SW 缓存版本升级

PWA Service Worker 缓存从 `claudio-v3` 升级到 `claudio-v4`，确保新版播放器结构和脚本生效。

### README 修正

- 心情检测时间说明从"跳过 09:00"修正为"跳过 9 点时段（09:00 与 09:40）"
- 补充自动续播说明：播放进入最后一首时后台自动调用 LLM 补歌
- `/api/refill` 说明从"队列耗尽时请求"改为"进入最后一首时后台请求新歌并追加播报与队列"

### 按钮响应修复

- 修复点击"今日歌单"后先卡住才显示"生成中"的问题
- 调整为先切换 UI 状态再做媒体解锁
