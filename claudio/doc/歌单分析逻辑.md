# Claudio 歌单分析逻辑

本文档介绍 Claudio 如何分析你的 `playlists.json` 和 `taste.md`，并结合 LLM 和网易云 API 实现个性化选曲。

---

## 总体流程

```
playlists.json ──┐
                 ├──→ [analyzer.js] ──→ taste_profile（缓存到 SQLite）
taste.md ────────┘                              │
                                                ↓
用户请求 / 定时任务 ──→ [context.js] 拼装 prompt ──→ [claude.js] 发给 LLM
                                                          │
                                                          ↓
                                                LLM 返回 { say, play, reason }
                                                          │
                                                          ↓
                                        [server.js] 歌曲搜索链路：
                                        ① matchFromPlaylists() 先在用户歌单里找
                                        ② search() 兜底全网搜索
                                        ③ getSongUrl() 获取可播放直链
                                        ④ savePlay() 写入 SQLite
                                        ⑤ broadcast() 推送到前端
```

---

## 第一条链路：歌单分析（analyzer.js）

这条链路在**服务启动时**自动执行，负责把 `playlists.json` 里的歌单转化成 AI 能理解的结构化信息。也可以通过 `POST /api/analyze` 手动触发刷新。

### 步骤 1：读取歌单配置

```js
function readPlaylistConfig() {
  const raw = readFileSync(join(rootDir, 'user', 'playlists.json'), 'utf-8');
  return JSON.parse(raw);
}
```

读取 `user/playlists.json` 中配置的所有歌单 ID，例如：

```json
{
  "uid": "6288667314",
  "playlists": [
    { "id": "17982800829", "name": "溜着玩" },
    { "id": "17982861814", "name": "kpop歌单" },
    ...
  ]
}
```

### 步骤 2：逐歌单拉取全部歌曲

对每个歌单 ID，同时做两件事：

- `getPlaylistAllTracks(id)` — 拉取歌单里所有歌曲（处理分页，每次 500 首）
- `getPlaylistInfo(id)` — 获取歌单名称、标签（tags）、描述

这一步调用 `ncm.js` 封装的网易云 API。

### 步骤 3：统计分析

对收集到的所有歌曲做以下分析：

| 分析维度 | 算法 |
|----------|------|
| 高频歌手 | 统计每个 artist 出现次数，取 Top 20 |
| 推断风格 | 根据歌单 tags 聚类（如 `K-pop`、`R&B`、`Hip-hop`） |
| 语言分布 | 检测歌曲名/歌手名的语言（中/英/日/韩） |
| 时段偏好 | 根据风格推断适合早中晚的推荐策略 |

### 步骤 4：生成 taste_profile 文本

分析结果被格式化为 Markdown 文本：

```markdown
# 歌单分析
共 4 个歌单，XXX 首不重复歌曲。

## 高频歌手
1. TWICE（42首）
2. 周杰伦（35首）
...

## 推断风格
K-pop为主，兼听华语R&B、日系City Pop

## 歌单歌曲抽样（供选曲参考）
- What is Love? - TWICE
- 晴天 - 周杰伦
- ...（最多 50 首）
```

该文本**缓存到 SQLite 的 `prefs` 表**中，键名为 `taste_profile`。同时还会缓存：

- `playlist_songs` — 最多 500 首歌曲详情（含 ID、歌名、歌手），用于后续 `matchFromPlaylists()` 匹配
- `taste_top_artists` — Top 20 歌手列表

### 步骤 5：自动回填 taste.md

`updateTasteFile()` 会读取 `taste.md`，如果某些章节为空（占位符），就用分析结果自动填充：

- `## 喜欢的歌手` ← 高频歌手 Top 15
- `## 偏好风格` ← 推断的风格描述
- `## 时段偏好` 各子项 ← 推断的时段推荐策略

> 如果 taste.md 已经手动写好了内容，auto-fill 不会覆盖已有内容。

## 第二条链路：AI 选曲（context.js → claude.js）

每次手动聊天或定时任务触发时，按以下流程走：

### 步骤 1：拼装 System Prompt

`buildContext()` 把 5 块内容拼成一条超长的 system prompt：

```
① dj-persona.md（DJ 人设）
    ↓ ---
② user/*.md（Taste.md 等手写偏好）
    ↓ ---
③ taste_profile（analyzer 缓存的分析结果 + 50首抽样歌单）
    ↓ ---
④ 当前环境（时间、天气、今日日历）
    ↓ ---
⑤ 最近播放 10 首（避免重复）
```

最终 System Prompt 大概是下面这样：

```
# Role
你是 Claudio，用户的私人 AI 电台 DJ。
你知道现在的时间、天气、用户的日程，以及他的音乐记忆。

# 性格
克制、有温度、懂得留白。
...

---

# 用户资料
# 音乐口味

## 偏好风格
K-pop（女团为主）、R&B、Hip-hop / Trap、华语流行与独立...

## 喜欢的歌手
TWICE、(G)I-DLE、KISS OF LIFE、NewJeans...

## 时段偏好
- 早晨：活力充沛的 K-pop 舞曲...
- 工作中：无人声或弱人声的后摇、电子氛围...
- 傍晚：放松的 City Pop...
- 睡前：极简钢琴...

---

# 歌单分析
共 4 个歌单，XXX 首不重复歌曲。

## 高频歌手
1. TWICE（42首）
2. 周杰伦（35首）
...

## 歌单歌曲抽样（供选曲参考）
- What is Love? - TWICE
- 晴天 - 周杰伦
...

---

# 当前环境
时间：2026/5/23 14:30:00
天气：晴 25°C
今日日程：无

---

# 最近播放（不要重复）
- LOVE DIVE - IVE（14:10）
- 七里香 - 周杰伦（13:50）
```

### 步骤 2：发给 LLM

调用 DeepSeek/OpenAI 兼容接口，LLM 返回严格 JSON：

```json
{
  "say": "下午两点的阳光斜进来，窗帘在风里晃了一圈...先让《What is Love?》进来",
  "play": ["What is Love? - TWICE", "晴天 - 周杰伦"],
  "reason": "用户偏好 K-pop 女团，下午适合轻快节奏",
  "segue": "换个心情"
}
```

**注意**：LLM 返回的 `play` 只是一个「歌名 - 歌手」的文本字符串，还不是可播放的歌曲。

### 步骤 3：歌曲搜索落地（两级匹配）

```
LLM 返回 "What is Love? - TWICE"
  │
  ├─ ① matchFromPlaylists("What is Love? - TWICE")
  │      └─ 在缓存好的 500 首用户歌单歌曲中模糊搜索
  │         （歌名匹配权重 10，歌手名匹配权重 5）
  │         → 命中！直接用 TWICE 在你的歌单里的那首
  │         → getSongUrl(id) 获取可播放直链
  │
  └─ ② 如果没命中 → search("What is Love? - TWICE", 1)
         └─ 全网搜索网易云曲库
         └─ getSongUrl(id) 获取可播放直链
```

**为什么优先在用户歌单里找？**

```js
export function matchFromPlaylists(query) {
  const songs = getPlaylistSongs();  // 从缓存读取 500 首
  const q = query.toLowerCase();
  const scored = songs
    .map(s => {
      const nameMatch = s.name.toLowerCase().includes(q);
      const artistMatch = s.artist.toLowerCase().includes(q);
      if (!nameMatch && !artistMatch) return { ...s, score: 0 };
      const score = (nameMatch ? 10 : 0) + (artistMatch ? 5 : 0);
      return { ...s, score };
    })
    .filter(s => s.score > 0)
    .sort((a, b) => b.score - a.score)
    .slice(0, 10);
  return scored;
}
```

- 歌名包含关键词 → +10 分
- 歌手名包含关键词 → +5 分
- 两者都匹配 → 15 分（最佳匹配）
- 取 Top 10 返回

这样做的好处：

- 优先播放你**自己歌单里已有的版本**（音质、版本都确定）
- 避免搜到翻唱版、live 版或非官方版本

## 歌曲搜索的 4 条统一链路

以下 4 个代码路径都使用完全相同的 NCM 搜索链路（`matchFromPlaylists → search → getSongUrl → savePlay → broadcast`）：

| 链路 | 文件 | 触发方式 |
|------|------|----------|
| 手动聊天 | `server.js` `/api/chat` | 用户在页面输入框发消息 |
| 今日歌单 | `server.js` `POST /api/plan/today` | 点击页面 📋 今日歌单 按钮 |
| 早报播报 | `scheduler.js` `planDay` | 每天 09:00 定时（10首） |
| 心情检测 | `scheduler.js` `moodCheck` | 每40分钟自动 |
| 自动续播 | `server.js` `/api/refill` | 播放队列播到最后一首时触发 |

## 缓存策略

| 缓存内容 | 存储位置 | 更新时机 |
|----------|----------|----------|
| `taste_profile`（分析报告） | SQLite `prefs` 表 | 服务启动 + `POST /api/analyze` 手动触发 |
| `playlist_songs`（500 首） | SQLite `prefs` 表 | 服务启动 + `POST /api/analyze` 手动触发 |
| `taste_top_artists`（Top 歌手） | SQLite `prefs` 表 | 服务启动 + `POST /api/analyze` 手动触发 |
| `today_plan`（今日播单） | SQLite `prefs` 表 | 每天 09:00 早报 + `POST /api/plan/today` 手动触发 |
| `today_events`（今日日历） | SQLite `prefs` 表 | 每天 08:55 预拉取 |

这些缓存避免了每次请求都重新拉取网易云 API（歌单数据量大，全量拉取耗时较长）。

## 相关文件

| 文件 | 作用 |
|------|------|
| `src/analyzer.js` | 歌单拉取、统计分析、taste_profile 生成、taste.md 自动回填 |
| `src/context.js` | 拼装 system prompt（人设 + taste.md + 分析结果 + 环境 + 播放历史） |
| `src/claude.js` | 调用 LLM API，返回 `say` / `play` / `reason` |
| `src/ncm.js` | 网易云 API 封装（搜索、歌单、URL 获取） |
| `server.js` | 歌曲落地链路：matchFromPlaylists → search → getSongUrl → savePlay → broadcast |
| `src/scheduler.js` | 定时任务（日历预拉取 08:55、早报 09:00、心情检测每40分钟） |
| `prompts/dj-persona.md` | DJ 人设 prompt（性格、播报词写法、输出格式） |
| `user/playlists.json` | 你配置的歌单 ID 列表 |
| `user/taste.md` | 你的音乐口味（手动编写 + analyzer 自动补充） |

## 设计思想

核心设计思想是「AI 负责决策，NCM API 负责执行」：

- **AI 负责决策** — 根据偏好 + 环境 + 心情，判断此刻该放什么歌
- **NCM API 负责执行** — 把 AI 输出的歌名落实为真实可播放的音频链接
- **matchFromPlaylists 做桥接** — 确保 AI 推荐的歌优先在你自己的歌单里找到

两者职责分离的结果是：AI 不需要知道网易云歌曲 ID，只需要理解你的口味；落地时由搜索链路保证一定能找到可播放的歌曲。
