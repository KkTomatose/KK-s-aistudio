# Claudio v1.3 更新说明

本文档汇总 v1.3 版本的所有新增功能、优化和修复。

---

## 1. 📋 今日歌单按钮 — 手动生成实时推荐

### 新增 `POST /api/plan/today` 端点

在聊天输入框右侧新增 **📋 今日歌单** 按钮，替代了原来仅作状态显示的 `CONNECTED` 标签。点击后根据**当前时间、实时天气、今日日历**即时生成 10 首推荐歌单。

**后端链路：**

```
getWeather() → getPref('today_events') → buildContext() → askClaude() → savePref('today_plan') → TTS synthesize → broadcast('say') + broadcast('now-playing') × 10
```

与 09:00 早报完全相同，**都会写入 `today_plan` 缓存**，区别仅在于：
- 不推送到飞书
- 每次点击都重新生成（虽是同 key 覆盖）
- 限制 10 首歌

### Bug 修复

- 修复 `savePref is not defined` — server.js 顶部 import 补上了 `savePref`
- 修复点击后不播报的问题 — 补上了 `synthesize()` TTS 合成调用

## 2. ⏰ 调度器调整

### 定时任务变更

| 任务 | v1.2 | v1.3 | 说明 |
|------|------|------|------|
| 歌单分析 | 每天 06:30 | **已删除** | 仅在服务启动时执行，手动可通过 `POST /api/analyze` 触发 |
| 日历预拉取 | 每天 06:55 | 每天 **08:55** | 配合早报时间调整 |
| 早报播报 | 每天 07:00 | 每天 **09:00** | 更合理的早间时间 |
| 工作开始推荐 | 每天 09:00 | **已删除** | — |
| 心情检测 | 每40分钟（跳过 07/09 点） | 每40分钟（仅跳过 **09** 点） | 07:00 取消后无需跳过 |

### 早报歌曲数量限制

早报和手动今日歌单统一限 10 首歌（`result.play.slice(0, 10)`），防止 LLM 返回过多歌曲。

## 3. 💾 页面 Session 持久化（按天隔离）

### 从「数据库回填」改为「localStorage 按天持久化」

**v1.2 行为：** 页面刷新 → `GET /api/history` → 从 SQLite 回填所有历史（含昨天数据）

**v1.3 行为：** 页面刷新 → 从 `localStorage` 精确恢复刷新前的状态 → WebSocket 实时追加

### 实现细节

- **Session Key 按日期生成：** `claudio-2026-05-23`，不同天的数据互不干扰
- **当天刷新：** 播放列表、聊天记录、播放进度精确还原
- **隔天打开：** localStorage key 日期变了，自动空白页面（全新一天）
- **同步写入：** 每次 `addToPlaylist`、`removeFromPlaylist`、`playNext`、收到 WebSocket 事件等操作都会即时写 localStorage

### 配套变化

- 删除了前端 `loadHistory()` 函数（不再从后端 `/api/history` 回填）
- 新增 `saveSession()` / `restoreSession()` / `getSessionKey()` 三个工具函数
- `messages` 数组追踪所有聊天消息，与 `playlist` 一起序列化

## 4. 🎵 播放列表交互优化

### 4a. 删除按钮改为 −（仅移除，不删数据库）

正面 QUEUE 列表每条歌曲右侧的 **× 按钮 → − 按钮**：

- **正面 −：** 仅从播放列表移除，**不删除数据库记录**
- **背面历史面板 ×：** 保持不变，仍然删除数据库记录

### 4b. 新增清空全部按钮

QUEUE 标题行右侧新增 **− 按钮**，一键清空所有歌曲：

- 停止当前播放
- 清空 `playlist` 数组
- 不接触数据库
- 同步更新 localStorage

### 4c. 点击行为优化

| 操作 | v1.2 | v1.3 |
|------|------|------|
| 点击当前播放的歌曲 | 重新从头播放 | 暂停/播放切换（复用顶部播放按钮逻辑） |
| 点击其他歌曲 | 切换到该歌曲 | 不变 |

## 5. 📂 折叠/展开功能

### QUEUE 播放列表折叠

点击 **QUEUE ▼** 标题行折叠/展开：
- 歌曲列表以动画收起（`max-height` + `opacity` 过渡）
- ▼ 箭头旋转 90° 变成 ▶

### 对话区域折叠

点击 **Claudio ▼ LIVE** 标题栏折叠/展开：
- 聊天消息区域以动画收起
- 折叠后仍可输入消息（输入框不受影响）

### CSS 修复

- `toggle-arrow` 统一 class，QUEUE 和对话共用
- 修复 `transform: rotate()` 不生效的问题 — `<span>` 默认 `display: inline` 不支持 transform，改为 `display: inline-block`

## 7. 🛡️ 歌单分析缓存保护

### 问题

服务启动时 `analyzePlaylists()` 自动执行，如果 NCM API 还没就绪导致拉取失败，旧代码会**无条件覆盖**已缓存的 `taste_profile`、`playlist_songs`、`taste_top_artists` 为空数据。

### 修复

拉取失败时先检查旧缓存是否存在：存在则**保留不动**并打印日志警告，不存在才写入空数据。

```
[analyzer] Failed to fetch any playlists. Keeping previous cache (2187 songs, 30 artists).
```
