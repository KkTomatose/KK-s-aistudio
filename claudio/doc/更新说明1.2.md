# Claudio v1.2 更新说明

本文档汇总 v1.2 版本的所有新增功能、优化和修复。

---

## 1. TTS 引擎可切换 + Fish Audio 集成

### 新增 `TTS_ENGINE` 环境变量

现在通过 `.env` 配置即可切换语音合成引擎，无需修改代码：

```env
TTS_ENGINE=edge    # 仅用 Edge TTS
TTS_ENGINE=fish    # 仅用 Fish Audio
TTS_ENGINE=auto    # 优先 Fish，失败自动回退 Edge（推荐）
```

### Fish Audio 配置

```env
FISH_API_KEY=你的 API Key
FISH_REFERENCE_ID=你的声音模型 ID
FISH_MODEL=s2-pro
```

- `FISH_REFERENCE_ID` 是 Fish Audio 的 voice model ID
- 获取方式：模型页 URL `https://fish.audio/m/<model_id>` 中的 `<model_id>` 部分
- `FISH_MODEL` 默认 `s2-pro`，可改为其他模型

### Auto 模式回退逻辑

```text
auto 模式:
  → 先尝试 Fish Audio
    → 成功 → 使用 Fish 生成的音频
    → 失败（网络超时 / 余额不足） → 自动回退 Edge TTS
  → 最终总能产出可播报音频
```

### Edge TTS 调用路径修复

修复了 Edge TTS 在本机的调用方式：

```diff
- uv run edge-tts --text "..." ...
+ uv run python -m edge_tts --text "..." ...
```

原因：`.venv/bin/edge-tts` 启动脚本的 shebang 指向旧项目路径，项目移动后失效。改为 `python -m edge_tts` 后不再依赖旧的 venv 脚本路径。

### Fish Audio 连通性说明

- Fish Audio 服务器在境外，中国大陆网络可能无法直连
- 使用全局 VPN / 系统代理后，Node.js 的 `fetch` 会自动走系统代理
- 需要充值才能使用（免费额度用完后返回 402 Insufficient Balance）

## 2. RECORD 按钮 + 卡片 3D 翻转

### 新增功能

页面顶部右上角新增 📀 **RECORD** 按钮，位于 SETTINGS 按钮左侧。

点击后：
- AI 电台卡片从正面 3D 翻转到背面
- **顶部 header 部分不变**（LOGO、主题切换、按钮）
- **正面 AI 电台内容不刷新、不丢失**
- 背面显示过往播放历史列表
- 再次点击 RECORD 翻回正面

### 过往播放列表

背面展示数据库中的最近 **200 条**播放记录：

- 样式与正面 QUEUE 歌曲列表完全一致（编号 + 歌名 + 歌手 + × 删除按钮）
- 列表可滚动

### 删除一首歌

点击 × → 同步删数据库 `DELETE /api/play/:dbId` → 自动从数据库补上一首，始终保持页面 200 条（除非数据库总量不足）。

### 清空全部

CLEAR ALL 按钮 → 删除当前页面所有记录 → 重新从数据库拉满 200 条补上。

### 3D 翻转滚动修复

3D 翻卡实现中，`perspective`、`transform-style: preserve-3d`、`backface-visibility` 等 CSS 3D 属性**仅在翻转状态下生效**。非翻转状态下零 3D 上下文，避免浏览器将整棵 DOM 子树合成为单层 GPU 纹理而导致滚动事件被拦截。

这解决了 Safari / Chrome 下队列列表和聊天区域的滚轮/触控板无法滚动的问题。

## 3. 播放队列播完自动续播

### 问题

之前播放队列全部播完后，播放器停住，不会自动请求新内容。

### 实现

新增 `POST /api/refill` 接口，完全复刻手动聊天的歌曲搜索链路。

**触发时机**：开始播放队列中最后一首歌时立即发起 refill 请求，而不是等到播完。

```text
最后一首歌开始
  → POST /api/refill
    → Claude 生成 DJ 播报 + 歌单
    → synthesize TTS 语音
    → matchFromPlaylists() → search() → getSongUrl() 搜索歌曲
    → broadcast('say') + broadcast('now-playing') × N
    → savePlay × N 写入 SQLite
  → 歌曲播放中，refill 在后台完成
  → 队列自然续上，无缝不间断
```

**异常情况**：若 refill 还没返回最后一首就播完了，前端会轮询等待新歌到达后自动切歌，不会停止。

### 防重保护

`isRefilling` 标志避免短时间内触发多次 refill 请求。

## 4. 定时任务 NCM 歌曲搜索修复

### 问题

调度器 `planDay`（早报/上午）和 `moodCheck`（心情检测）在 v1.1 中加入 `savePlay` 后，直接将 Claude 返回的原始歌名写入数据库，没有经过网易云搜索。导致：

- `song_name` 列存的是原始文本（非标准歌名）
- `artist` 列为空
- `ncm_id` 为空
- `url` 为 null → 前端 `playAudio` 检查 `if (!url) return` → 无法播放

### 修复

三条歌曲写入链路现已完全统一：

```text
Claude 返回歌名
  → matchFromPlaylists(歌名)
    → 命中 → getSongUrl(id) → url
    → 未命中 → search(歌名, 1) → getSongUrl(id) → url
      → savePlay(song.name, song.artist, song.id, reason)
      → broadcast('now-playing', { name, artist, url, id, dbId })
```

涉及文件：

| 链路 | 文件 |
|---|---|
| 手动聊天 | [server.js](file:///Users/a/Desktop/OH-WorkSpace/ALL/我的Github项目/claudio/server.js) `/api/chat` |
| 早报 / 上午 | [scheduler.js](file:///Users/a/Desktop/OH-WorkSpace/ALL/我的Github项目/claudio/src/scheduler.js) `planDay` |
| 心情检测 | [scheduler.js](file:///Users/a/Desktop/OH-WorkSpace/ALL/我的Github项目/claudio/src/scheduler.js) `moodCheck` |

## 5. 心情检测间隔调整

```diff
- schedule.scheduleJob('0 * * * *', ...)   // 每小时整点
+ schedule.scheduleJob('0,40 * * * *', ...) // 每小时 0 分和 40 分（≈每 40 分钟）
```

仍然跳过 7:00 和 9:00（这两个时段有独立播报）。

### 心情检测行为说明

心情检测不是每次都播报——它查最近 3 首播放记录发给 Claude，只有 Claude 判断适合换风格并推荐了歌时，才会：

- 在聊天区显示 DJ 气泡文字（`broadcast('say', { text })`）
- 向队列推送歌曲（`broadcast('now-playing', ...)`）

当前心情检测只发文字气泡，**不合成 TTS 语音播报**。

## 6. 播放控制按钮说明

### 暂停 vs 停止

| | 暂停（⏯️） | 停止（⬛） |
|---|---|---|
| 音频 | 暂停 | 暂停 |
| 进度 | 保留当前位置 | 归零到开头 |
| 再按播放 | 从暂停处继续 | 从头播放同一首 |
| 曲目标题 | 不变 | 不变 |
| 清除 src | 否 | 否 |

`resetPlayer`：只在队列被清空时自动调用，把播放器界面恢复到默认 "Claudio" 状态，没有手动按钮触发。

- 修改 `.env` 后需**重启 Node 服务**
- `TTS_ENGINE=auto` 在 Fish 网络不通时会每次等超时再回退，若确定无法使用 Fish，建议改为 `TTS_ENGINE=edge`

- 定时任务只在 Node 进程运行期间生效，关掉终端即停止
